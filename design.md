# 喜人同乐 Vue3 复刻设计说明

## 目标

基于上传的 React 项目 `喜人同乐.zip`，在不改变信息架构、页面内容、核心交互和视觉方向的前提下，复刻一个 Vue3 前端项目。

本次实现遵循：

- 路由结构与 React 版本保持一致
- 静态文案、榜单数据、文章内容、作者数据按原项目迁移
- 弹窗、筛选、评论、点赞、收藏、关注等交互按原项目逻辑模拟
- 不新增原项目中没有的页面和业务流程

## 技术栈

- Vue 3
- Vite
- Vue Router
- Pinia
- Axios
- Element Plus

说明：

- Element Plus 已完成全局接入，用于满足指定技术栈要求
- 为保持页面观感与原 React 版一致，页面主体样式以自定义 CSS 为主，没有强行替换成 Element Plus 默认视觉

## 路由映射

与 React 原项目一一对应：

- `/`：首页
- `/traffic`：流量页
- `/traffic/youtube`：YouTube 工具集
- `/traffic/youtube/channels`：频道榜单
- `/traffic/youtube/trending`：爆款视频追踪
- `/traffic/youtube/rising`：潜力新账号
- `/article/:id`：文章详情
- `/author/:name`：作者主页
- `/register`：注册页
- `/forgot-password`：忘记密码页

## 结构设计

### 1. 布局层

- `src/layouts/SiteLayout.vue`
- 负责顶部导航、首页二级分类导航、底部版权、登录弹窗挂载
- 首页二级导航保留原 React 版的滚动显隐逻辑

### 2. 共享组件

- `SiteLogo.vue`：站点 Logo
- `BaseAvatar.vue`：统一头像组件
- `LoginDialog.vue`：登录弹窗
- `LanguageSelect.vue`：榜单语言下拉
- `IconByName.vue`：将字符串图标名映射为 `lucide-vue-next` 图标

### 3. 状态管理

- `src/stores/ui.ts`
- 使用 Pinia 承载：
  - 登录弹窗开关
  - 首页当前分类

## 数据设计

为避免迁移过程中内容偏移，静态数据从页面中抽离为独立模块：

- `src/data/articles.ts`
  - 首页推荐卡片
  - 公告
  - 热门文章
  - 文章列表
  - 相关文章标题
- `src/data/traffic.ts`
  - 流量页平台矩阵
  - YouTube 工具集配置
- `src/data/youtube.ts`
  - 频道榜单
  - 爆款视频追踪
  - 潜力新账号
  - 分类与语言选项
- `src/data/authors.ts`
  - 作者资料、成就、历史数据

## 页面复刻说明

### 首页

- 保留橙色 Hero 区和 4 张推荐卡片
- 左侧文章流、右侧公告/活动/热门/快捷入口布局保持不变
- 分类筛选规则与原版一致：
  - `全部` 显示全部文章
  - 其他分类按 `category` 或 `tags` 匹配

### 流量页

- 海外平台、国内平台、跨平台三个分区保持一致
- 平台卡片的图标、badge、描述、跳转规则与原版对齐

### YouTube 工具页

- 工具卡片顺序与 React 版一致
- 已开通的工具保留可点击跳转

### 榜单页与追踪页

- 频道榜单：分类筛选、排名展示、频道信息结构保持一致
- 爆款视频追踪：时间切换、语言下拉、视频增长指标保持一致
- 潜力新账号：语言筛选、增长 badge、分类 tag、统计指标保持一致

### 文章详情页

- 保留：
  - 左侧悬浮操作按钮
  - 作者信息区
  - 正文渲染
  - 点赞区
  - 评论输入与评论列表
  - 右侧作者侧栏、目录、PDF、相关文章
- 文章正文按原 React 版的轻量 Markdown 规则解析：
  - `#` / `##` 标题
  - 列表
  - `**加粗**`

### 作者主页

- 保留顶部橙色渐变 Hero 和叠层作者卡片
- 文章 / 关注 / 收藏三个 Tab 与原版一致
- 关注按钮会同步修改粉丝数

### 注册页 / 忘记密码页

- 表单校验逻辑保持原版文案和交互
- 成功状态页保持原版结构和按钮行为

## 样式策略

- 不引入 Tailwind，改为自定义 CSS 复刻布局和视觉
- 全局变量、基础样式和通用类位于 `src/styles/index.css`
- 各页面使用 SFC scoped CSS 局部控制细节
- 颜色、圆角、间距、文字层级尽量按 React 原实现复用

## Axios 设计

- `src/api/http.ts` 中已创建 Axios 实例
- 当前项目与原 React 版一样，核心展示数据以前端静态数据为主
- Axios 作为后续接真实接口的统一入口，不主动虚构后端请求

## 复刻边界

本项目有意保持“前端静态演示站”属性，不额外新增：

- 登录注册真实接口
- 评论持久化
- 点赞收藏持久化
- 搜索结果页
- 新的管理页或营销页

如后续需要继续扩展，可以在现有数据模块和 Axios 入口基础上接真实后端，而无需重构页面骨架。
